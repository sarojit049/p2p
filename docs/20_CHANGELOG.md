# Versioning Standard

PrivateConnect follows Semantic Versioning.

Format
```text
MAJOR.MINOR.PATCH
```

Examples
```text
1.0.0
1.0.1
1.1.0
2.0.0
```

Definitions

MAJOR
Breaking changes

MINOR
New backward-compatible features

PATCH
Bug fixes

# Release Workflow

Before every release:
1. Update version number.
2. Update project progress.
3. Complete testing.
4. Update documentation.
5. Tag release in Git.
6. Deploy application.
7. Verify smoke tests.
